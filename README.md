# Log Analyzer (Анализатор логов веб-сервера)

Десктопное приложение для парсинга и анализа файлов журналов доступа (access logs) веб-серверов. Разработано в рамках курсовой работы.

Программа умеет обрабатывать большие объемы логов (Nginx, Apache), строить графики активности, выявлять подозрительные IP-адреса и экспортировать результаты аналитики.

## ✨ Основной функционал

* **Интеллектуальный парсинг:** Автоматическое определение формата логов (Apache Common, Apache Combined, Nginx).
* **Сводная статистика:** Подсчет общего количества запросов, уникальных IP-адресов и доли ошибок (4xx, 5xx).
* **Визуализация:** Почасовой график распределения трафика и круговая диаграмма кодов ответа сервера.
* **Анализ безопасности:** Автоматическое выявление подозрительных IP-адресов на основе настраиваемого порога частоты запросов.
* **Поиск и фильтрация:** Быстрый поиск по IP-адресам, URL-страницам и журналу ошибок.
* **Экспорт:** Выгрузка отчетов в формате `.csv` для дальнейшей работы в Excel.

## 🛠 Стек технологий

## 🚀 Инструкция по запуску

Так как графический интерфейс написан на WinForms, приложение предназначено **только для ОС Windows**.

### Системные требования:
* OS Windows (10/11)
* Установленный [.NET 8.0 SDK](https://dotnet.microsoft.com/download/dotnet/8.0)
* C# 8+

### Вариант 1. Запуск через Visual Studio (Рекомендуется)
1. Склонируйте репозиторий или распакуйте архив с проектом.
2. Откройте файл решения `KR_Algorithms_LogResearcher.sln` в Visual Studio 2022.
3. Убедитесь, что в панели обозревателя решений (Solution Explorer) проект **KR_Algorithms_LogResearcher** назначен как запускаемый по умолчанию (выделен жирным шрифтом). Если нет — нажмите на него правой кнопкой мыши и выберите *"Set as Startup Project"*.
4. Нажмите `F5` или кнопку **"Start"** (Запуск) на верхней панели.

### Вариант 2. Запуск через командную строку (CLI)
1. Откройте командную строку (cmd) или PowerShell.
2. Перейдите в корневую папку проекта (где лежит файл `.sln`).
3. Выполните сборку проекта:
   ```bash
   dotnet build
   ```
4. Запустите приложение:
   ```bash
   dotnet run --project KR_Algorithms_LogResearcher\KR_Algorithms_LogResearcher.csproj
   ```

## 📖 Как пользоваться программой

1. Нажмите кнопку **"📂 Открыть"** на верхней панели инструментов и выберите файл логов (`.log` или `.txt`).
2. При необходимости в правом верхнем углу измените **Порог** — количество запросов с одного IP, после которого система пометит его как "Подозрительный".
3. Нажмите кнопку **"▶ Анализ"**. Дождитесь заполнения прогресс-бара.
4. Переключайтесь между вкладками для просмотра данных:
   * **Обзор:** Графики и общая статистика.
   * **IP адреса:** Детальная таблица по посетителям и выявление ботов.
   * **Страницы:** Популярность маршрутов и наличие битых ссылок.
   * **Ошибки:** Журнал неудачных запросов.
5. Для выгрузки данных используйте кнопку **"📥 Экспорт CSV"** на нужной вкладке.

## 🧪 Запуск тестов

Проект покрыт unit-тестами (xUnit), которые проверяют корректность расчетов, обработку "битых" строк в логах и логику поиска.

Для запуска тестов через командную строку выполните:
```bash
dotnet test
```
*Либо используйте встроенный Test Explorer в Visual Studio (меню Test -> Run All Tests).*

## 📁 Структура проекта

* `KR_Algorithms_LogResearcher/` — Главный проект с графическим интерфейсом (WinForms).
* `Controllers/` — Бизнес-логика проекта. Включает ядро приложения `LogController`, отвечающее за потоковое чтение, парсинг регулярными выражениями и агрегацию данных.
* `Models/` — Структуры данных (`LogEntry`, `IPStatistics`, `PageStatistics`).
* `KR_Algorithms_LogResearcher.Tests/` — Проект с xUnit тестами.
